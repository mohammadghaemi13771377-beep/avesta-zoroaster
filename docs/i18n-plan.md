# برنامه چندزبانه AVESTA-ZOROASTER

هدف چندزبانه‌سازی فقط ترجمه منو نیست. سایت باید بتواند برای هر متن اوستایی، مقاله، واژه‌نامه و منبع کتابخانه‌ای نسخه فارسی و انگلیسی مستقل داشته باشد، بدون اینکه ساختار اصلی محتوا دو بار ساخته شود.

## زبان‌های فاز اول

- فارسی: زبان اصلی، RTL و مسیر پیش‌فرض
- انگلیسی: زبان دوم، LTR و مسیر ورودی `/en`

## معماری فعلی

- `lib/i18n.ts` زبان‌ها، تنظیمات جهت متن، دیکشنری UI و helper لینک را نگه می‌دارد.
- `/fa` و `/en` صفحه ورودی محلی‌سازی‌شده دارند.
- `/api/i18n` تنظیمات زبان را برمی‌گرداند.
- `/api/i18n/dictionary?locale=en` دیکشنری UI همان زبان را برمی‌گرداند.
- Prisma برای محتوای بلند مدل‌های Translation جدا دارد.

## مدل‌های ترجمه در دیتابیس

- `AvestaSectionTranslation`
- `AvestaChapterTranslation`
- `AvestaVerseTranslation`
- `ArticleTranslation`
- `GlossaryTermTranslation`
- `LibraryItemTranslation`

این مدل‌ها با enum `Locale` کار می‌کنند و برای هر آیتم، هر زبان فقط یک نسخه ترجمه دارد.

## قانون تولید صفحه

- فارسی به عنوان canonical اصلی استفاده می‌شود.
- انگلیسی برای صفحه ورودی با `/en` شروع می‌شود.
- تا وقتی همه مسیرها localized نشده‌اند، لینک‌های انگلیسی به مسیرهای اصلی با `?lang=en` می‌روند تا کاربر به 404 نخورد.
- در مرحله بعدی، مسیرهای کامل مثل `/en/avesta/yasna` با shared route handler ساخته می‌شوند.

## اولویت مرحله بعد

1. اتصال صفحات محتوایی به Prisma و خواندن translation براساس `lang`.
2. افزودن alternate metadata برای صفحات اوستا، مقاله و واژه‌نامه.
3. افزودن فیلتر زبان به Meilisearch.
4. ساخت فرم ادمین برای وارد کردن ترجمه هر محتوا.
5. ایجاد مسیرهای کامل `/en/...` برای صفحات اصلی.

## استاندارد محتوا

برای هر بند اوستا، زبان‌ها باید جدا اما هم‌ساختار باشند:

- متن اصلی
- آوانویسی
- ترجمه کلاسیک
- بازنویسی ساده
- تحلیل مفهومی
- پیام اخلاقی

این ساختار باعث می‌شود سایت هم دانشنامه‌ای باشد، هم قابل مطالعه، هم آماده رشد برای هزاران صفحه.
