# جریان داده اوستا

این مرحله یک لایه میانی برای خواندن محتوای اوستا اضافه می‌کند تا صفحه‌ها مستقیم به داده نمایشی قفل نشوند.

## فایل اصلی

`lib/avesta-repository.ts`

این فایل سه کار انجام می‌دهد:

- اگر Prisma و دیتابیس آماده باشند، محتوا را از PostgreSQL می‌خواند.
- اگر دیتابیس هنوز آماده نباشد، از محتوای نمونه در `lib/sample-content.ts` و `lib/content.ts` استفاده می‌کند.
- زبان را از `lang=fa` یا `lang=en` می‌گیرد و در صورت وجود translation همان زبان را برمی‌گرداند.

## صفحه‌های وصل‌شده

- `/avesta`
- `/avesta/[section]`
- `/avesta/[section]/[chapter]/[verse]`

## APIهای خواندن اوستا

- `/api/avesta`
- `/api/avesta?lang=en`
- `/api/avesta/yasna`
- `/api/avesta/yasna?lang=en`
- `/api/avesta/yasna/ha-1/verse-1`
- `/api/avesta/yasna/ha-1/verse-1?lang=en`

در endpoint بند، پاسخ علاوه بر `verse` شامل `media` هم هست؛ یعنی تصویر AI، صوت یا prompt مرتبط با همان بند از `MediaAsset` خوانده می‌شود.

## مسیر مهاجرت به دیتابیس واقعی

1. اجرای migration با Prisma
2. اجرای seed اولیه
3. وارد کردن متن کامل اوستا از پنل ادمین یا اسکریپت import
4. ثبت رسانه‌های مرتبط در `MediaAsset`
5. جایگزین کردن fallback نمونه با داده واقعی
6. فعال کردن sync به Meilisearch برای جستجوی سریع

## مزیت معماری

صفحه‌ها، APIها و پنل آینده از یک منبع واحد می‌خوانند. بنابراین وقتی محتوا وارد دیتابیس شود، هم سایت، هم جستجو، هم اپ موبایل آینده بدون بازنویسی بزرگ به‌روز می‌شوند.
