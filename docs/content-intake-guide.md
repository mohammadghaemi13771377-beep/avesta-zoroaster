# راهنمای ورود محتوا و تصویر

برای این پروژه لازم نیست همه محتوا و تصویرها از روز اول داخل کد باشند. ساختار فعلی طوری آماده شده که بعداً بتوانی محتوا را از پنل ادمین، دیتابیس یا فایل‌های آماده وارد کنی.

## آیا تصویر AI واقعی بارگذاری شده؟

فعلاً نه. در کد مسیرها، promptها، مدل دیتابیس و جایگاه نمایش آماده شده‌اند، اما فایل واقعی تصویر باید بعداً در مسیرهای `public/images/ai` قرار بگیرد یا از طریق storage آپلود شود.

## مسیرهای پیشنهادی فایل‌ها

- تصویرهای AI: `public/images/ai`
- صوت‌ها: `public/audio`
- PDFها: `public/library`
- ویدیوها: `public/video`

## نمونه نام‌گذاری

- `home-hero-sunrise.jpg`
- `yasna-cover.jpg`
- `gathas-cover.jpg`
- `vendidad-cover.jpg`
- `yasna-ha-1-verse-1.jpg`
- `yasna-verse-1.mp3`
- `avesta-reading-guide.pdf`

## اگر بعداً خودت خواستی پر کنی

1. وارد `/admin/media` شو.
2. در فرم «آپلود فایل» نوع فایل را انتخاب کن.
3. فایل را آپلود کن و URL خروجی را بردار.
4. URL را در فرم «ثبت رسانه» بگذار، مثل:

```txt
/images/ai/yasna-cover.jpg
```

5. اگر رسانه مربوط به بند خاصی است، `sectionSlug`، `chapterSlug` و `verseOrder` را پر کن.

در حالت توسعه لوکال، فایل‌ها داخل پوشه `public` ذخیره می‌شوند. در deploy واقعی بهتر است همین API به storage دائمی مثل S3، Cloudflare R2 یا فضای ذخیره‌سازی سرور وصل شود.

## API آپلود

```http
POST /api/admin/upload
Content-Type: multipart/form-data
```

فیلدها:

- `file`: فایل واقعی
- `kind`: یکی از `image`، `audio`، `pdf`، `video`
- `name`: نام پیشنهادی فایل

## اگر محتوا را اینجا برای دان بفرستی

بهترین قالب برای ارسال محتوا:

```txt
بخش: yasna
فصل: ha-1
بند: 1
عنوان تصویر: آتش مقدس یسنا
مسیر پیشنهادی: /images/ai/yasna-ha-1-verse-1.jpg
متن اصلی:
...
ترجمه کلاسیک:
...
بازنویسی ساده:
...
تحلیل مفهومی:
...
پیام اخلاقی:
...
prompt تصویر:
...
```

با این قالب، محتوا مستقیم به مدل‌های فعلی پروژه می‌نشیند.

## ورود دسته‌ای

اگر محتوا زیاد باشد، مسیر بهتر استفاده از import دسته‌ای است:

```txt
/admin/import
```

یا از API:

```http
POST /api/admin/import
GET /api/admin/import/{id}
```

قالب کامل در `docs/bulk-import-guide.md` آمده است.

## جایگاه‌های آماده

API زیر فهرست جایگاه‌های آماده را می‌دهد:

```http
GET /api/admin/slots
```

همین لیست در پنل `/admin/media` هم نمایش داده می‌شود.
