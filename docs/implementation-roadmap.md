# نقشه اجرای فنی AVESTA-ZOROASTER

## فاز ۱: تجربه اولیه

وضعیت: انجام شده

- اسکلت Next.js، TypeScript و Tailwind
- RTL فارسی
- صفحه خانه سینمایی
- پورتال `/avesta`
- صفحات پایه مسیرها
- مدل Prisma
- جستجوی نمونه
- واژه‌نامه نمونه
- حالت مطالعه نمونه
- تایم‌لاین تاریخی
- فرم نمونه پنل مدیریت
- API نمونه ذخیره محتوا
- اکسپلورر کتابخانه با فیلتر
- گالری رسانه و promptهای تصویرسازی AI
- صفحه مقالات با فیلتر، تگ و دسته‌بندی
- صفحه جزئیات مقاله با JSON-LD و لینک به واژه‌نامه و متن اوستا
- هاب یکتاپرستی با ستون‌های اهورامزدا، اشا، اختیار انسان و اخلاق سه‌گانه
- بازطراحی صفحه دین زرتشتی به عنوان مسیر ورود به یکتاپرستی، گات‌ها، واژه‌نامه و تایم‌لاین
- پروفایل مطالعه با ادامه مطالعه، بوکمارک‌ها، آمار و تنظیمات مطالعه
- مدل‌های Prisma برای User، Bookmark، ReadingProgress و ReadingPreference
- API نمونه برای بوکمارک و پیشرفت مطالعه
- ورود و ثبت‌نام demo با session cookie
- middleware محافظ برای `/profile` و `/admin`
- داشبورد ادمین با sidebar، آمار محتوا، صف انتشار، وضعیت SEO و سلامت سیستم
- زیرصفحه‌های ادمین برای اوستا، مقاله‌ها، واژه‌نامه، رسانه و SEO
- معماری جستجوی چند index برای Meilisearch
- APIهای search indexes و search sync در حالت dry-run

## فاز ۲: دیتابیس و محتوا

1. نصب PostgreSQL
2. تنظیم `DATABASE_URL`
3. اجرای `npx prisma migrate dev`
4. ساخت seed برای AvestaSection، AvestaChapter، AvestaVerse
5. اتصال صفحات اوستا به Prisma
6. ساخت upload pipeline برای تصویر AI و صوت

## فاز ۳: پنل مدیریت

گزینه سریع:

- Strapi CMS
- اتصال مدل‌ها به API
- مدیریت تصویر، صوت، SEO و دسته‌بندی‌ها

گزینه اختصاصی:

- مسیر `/admin`
- احراز هویت
- CRUD کامل برای متن‌ها، مقاله‌ها، واژه‌نامه و کتابخانه

## فاز ۴: جستجوی واقعی

1. نصب Meilisearch
2. ساخت indexهای `verses`، `articles`، `glossary`، `library`
3. sync دیتابیس با Meilisearch
4. فیلتر بر اساس نوع محتوا، بخش اوستا، زبان و تگ
